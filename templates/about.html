🧐 About 🧐

[mini-blog] è un'applicazione Flask basata su un tutorial disponibile su YouTube. Il tutorial si trova al seguente link: YouTube Tutorial. Inoltre, il sorgente originale è disponibile su GitHub all'indirizzo: GitHub - FlaskIntroduction.

Ho modificato il file app.py con le seguenti modifiche:

  • Aggiunta di app.config["SQLALCHEMY_TRACK_MODIFICATIONS"] = False
  • Quando è impostato su False, SQLAlchemy smette di tenere traccia automaticamente delle modifiche alle istanze degli oggetti, riducendo così il carico sulle risorse del sistema e migliorando le prestazioni dell'applicazione.
  • Definizione della classe Todo con il campo title e agginto colonna nel db nel modello
  • Aggiunta di nuove route:
    • @app.route("/home") - Reindirizzamento alla home
    • @app.route("/about") - Pagina "About"
    • @app.route("/", methods=["POST", "GET"]) - Gestione dei task e dei nuovi contenuti
    • @app.route("/update/<int:id>", methods=["GET", "POST"]) - Aggiornamento dei task

Inoltre, sono stati apportati miglioramenti al codice HTML e sono state aggiunte modifiche al file main.css per migliorare lo stile dell'applicazione.

All'interno di un documento HTML, gli stili definiti direttamente all'interno del tag <style> nell'intestazione hanno una priorità maggiore rispetto agli stili definiti in un file CSS separato, come ad esempio main.css. Questo concetto si basa sulla specificità dei selettori CSS e sull'ordine di applicazione delle regole di stile. Gli stili inline, definiti direttamente negli attributi degli elementi HTML (ad esempio, <p style="color: red;">Testo rosso</p>), hanno la massima priorità e sovrascrivono qualsiasi altra regola CSS. In caso di conflitto tra stili, la regola CSS con maggiore specificità viene applicata. Ad esempio, uno stile definito per un ID (#myElement) avrà più peso rispetto a uno stile definito per una classe (.myClass) o per un elemento specifico (p). Inoltre, l'ultima regola dichiarata viene applicata nel caso in cui più regole CSS abbiano la stessa specificità.
